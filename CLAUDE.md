# **必ず守るルール**
- ユーザーから全機能の実装を指示された場合は、全機能に対して `/add-feature` コマンドを実行し、途中で作業を一切止めずに全機能を実装しきること。
- `/add-feature` コマンドの内容を実行し終わったタイミングで毎回、
  - コンテキストが失われないように次のファイルを読み込むこと。
    - @docs/ 配下の、ideasディレクトリを除く全ファイル
    - 実装中機能のステアリングファイル
  - Codexへコードレビューを依頼すること。
  - コードレビューでの指摘内容が完全になくなるまで、修正とコミット、Codexへの再レビュー依頼を出すこと。
  - コードレビューが終わるたびに、 @CLAUDE.md を読み込むこと。
- 作業用ブランチは指示がなければmainから作成し、そこで作業。コミット。プッシュを行い、最後にmainへ向けてPRを作成すること。
- GitHub Issue作成時は、必要に応じてラベルをつける。
- GitHub Issueを元に開発を行う場合は、作業用ブランチを最新のmainから作成する。また、PR作成時は該当Issueと必ず紐づける。
- `*.py` ファイルを編集したら、必ずリント（`uv run ruff check src/ tests/`）と型チェック（`uv run mypy src/`）を実行し、エラーがあれば修正すること。
- @.gitignore に記載されているディレクトリ/ファイルは絶対にコミットしない。
